# 회의록 작성 - $ARGUMENTS

회의 녹취록을 분석하여 체계적인 회의록 마크다운 문서로 생성합니다.

**첫 번째 인자**: 날짜 (YYYY-MM-DD 형식, 선택사항 - 생략시 오늘 날짜 사용)
**두 번째 인자**: 회의 녹취록 내용

## 작업 프로세스

1. **인자 파싱 및 녹취록 분석**

   - 첫 번째 인자가 YYYY-MM-DD 형식인지 확인
   - 날짜 형식이면 해당 날짜 사용, 아니면 오늘 날짜 사용
   - 녹취록에서 회의 정보 추출:
     - 회의 제목/주제 식별
     - 참석자 파악 (발언자 이름 기반)
     - 안건별 논의 내용 분류
     - 결정 사항 및 합의 내용 추출
     - 액션 아이템과 담당자 식별
     - 배경 정보 및 맥락 파악
     - 위험 요소 및 우려사항 식별
     - 다음 회의 일정 또는 후속 조치 확인

2. **마크다운 문서 생성**

   - 파일명: `YYYY-MM-DD-회의명.md` (지정된 날짜 또는 오늘 날짜 기반)
   - ex: `2025-12-02-신규프로젝트-킥오프.md`
   - 저장 위치: `~/DocumentsLocal/msbaek_vault/work-log/`
   - 섹션별로 체계적으로 구조화
   - 선택적 섹션(위험요소, 다음 회의)은 해당 내용이 있을 경우에만 포함

## 처리 로직

### 1단계: 인자 파싱 및 녹취록 분석

```
입력된 $ARGUMENTS 파싱:
1. 첫 번째 인자가 YYYY-MM-DD 형식인지 정규식으로 확인
2. 날짜 형식이면:
   - 날짜 = 첫 번째 인자
   - 녹취록 = 나머지 인자들 결합
3. 날짜 형식이 아니면:
   - 날짜 = 오늘 날짜 (date +"%Y-%m-%d")
   - 녹취록 = 모든 인자들 결합

녹취록 분석:
- 발언자 패턴 식별 (이름:, [이름], 이름님 등)
- 회의 주제 및 목적 파악
- 안건별로 논의 내용 그룹화
- "결정", "합의", "확정" 등의 키워드로 결정사항 추출
- "해야", "하기로", "담당" 등의 키워드로 액션아이템 추출
- "우려", "위험", "문제" 등의 키워드로 위험요소 추출
- "다음 회의", "다음 주" 등의 키워드로 후속 일정 추출
```

### 2단계: 문서 생성

```
파싱된 날짜를 기준으로:
- 파일명: YYYY-MM-DD-회의명.md (회의명은 녹취록에서 추출, 공백은 하이픈으로 대체)
- ~/DocumentsLocal/msbaek_vault/work-log/ 디렉토리에 저장
- 체계적인 마크다운 형식으로 구조화
- 문서 제목에는 한국어 날짜 형식 사용 (YYYY년 MM월 DD일)
- 표 형식으로 결정사항과 액션아이템 정리
```

## 주의사항

- 저장 위치는 `~/DocumentsLocal/msbaek_vault/work-log/`
- 회의명은 녹취록에서 추출하여 파일명에 사용 (공백은 하이픈으로 대체)
- 같은 파일명이 이미 존재하는 경우, 파일명에 순번 추가 (예: `2025-12-02-신규프로젝트-킥오프-2.md`)
- 녹취록이 비어있거나 너무 짧으면 경고 메시지 출력
- 발언자 구분이 불명확한 경우 "참석자 미상"으로 표기
- Obsidian vault에 저장되므로 Git 작업은 별도로 필요 없음
